(n, k)-Local Polynomial Simplicial Attention

June 2026

Softmax attention is local constant regression on tokens. What if the datapoints are not tokens but pairs or triples of tokens + what if you fit something richer than a constant?

Attention is a regression problem over the key-value pairs in context [1]. Softmax is the local constant estimator (Nadaraya-Watson), LLA [2] upgrades it to a local linear fit, and our recent work on Parallax [3] replaces LLA's per-token solve with a learned probe into the local KV covariance. In this work we will generalize along the axes of the fit and the data.

The first axis is simplex order $n$. It decides whether a datapoint in our regression problem is a token, a token-pair, a triple, or an $n$-tuple. The second axis is regression degree $k$. It decides whether you fit a local constant, a local linear fit, a local quadratic fit, and so on. Below we examine the cross-product of these two choices.

Lifting attention onto simplices

Start with the $n$ axis and leave $k = 0$. Fix a query $i$ and let $J = (j_1, \dots, j_n)$ index an $n$-tuple of context tokens. The set of admissible tuples $\mathcal{C}_i^{(n)}$ is the simplicial memory, with the query sitting as the extra vertex.

Each tuple needs one key and one value, and so we lift the per-token keys and values through a map.

$$\phi_J = \Phi\big(k^{(1)}_{j_1}, \dots, k^{(n)}_{j_n}\big), \qquad u_J = U\big(v^{(1)}_{j_1}, \dots, v^{(n)}_{j_n}\big)$$

The cheap version takes Hadamard products instead of a general tensor map, though there are many other choices of admissible map.

$$\phi_J = k^{(1)}_{j_1} \odot \cdots \odot k^{(n)}_{j_n}, \qquad u_J = v^{(1)}_{j_1} \odot \cdots \odot v^{(n)}_{j_n}$$

This diagonal lift is a specific (convenient) choice and is used in the "Fast and Simplex" paper [4]. It keeps the logit an ordinary dot product in lifted space, and for pairs becomes the trilinear score.

$$s_{iJ} = \psi_i^\top \phi_J, \qquad \psi_i^\top \phi_{jk} = \sum_{\ell} q_{i\ell}\, k_{j\ell}\, k'_{k\ell}$$

The output is then computed by running softmax over the lifted dataset taking a kernel-weighted average of the tuple values. We have now moved to local constant regression over lifted simplices.

$$o_i^{(n,0)} = \mathbb{E}_{p_i}[u_J], \qquad p_{iJ} = \frac{\exp(\psi_i^\top \phi_J / h)}{\sum_{J'} \exp(\psi_i^\top \phi_{J'} / h)}$$

For $n = 1$ this is ordinary softmax attention. For $n = 2$ it is 2-simplicial attention [5], and so on. We haven't changed the regression problem, only the data.

From local constant to local polynomial

Now hold $n$ and turn the $k$ knob. Softmax fits a constant to the local cloud of tuples, but we can fit a polynomial instead and read it off at the query.

We center the lifted features at the query and build monomial features up to degree $k$.

$$z_J = \phi_J - \psi_i, \qquad \chi_k(z) = \big[\,1,\; z,\; z^{\otimes 2},\; \dots,\; z^{\otimes k}\,\big]$$

The exact $(n,k)$ estimator is a weighted least squares fit of a degree-$k$ polynomial over the simplex dataset, with an optional regularizer.

$$\hat P_i = \arg\min_{P \in \Pi_k} \sum_{J \in \mathcal{C}_i^{(n)}} w_{iJ}\,\big\| P(\phi_J - \psi_i) - u_J \big\|^2 + \Omega(P)$$

The output is that polynomial evaluated at the query center, where $z = 0$, aka the intercept.

$$o_i^{(n,k)} = \hat P_i(0)$$

$(n,k)$-local polynomial simplicial attention is simply local polynomial regression over lifted simplex datapoints.

The equivalent-kernel view

The fit has a closed form, but it looks quite different from softmax weights. We stack the monomial features and form the weighted Gram matrix, then solve the normal equations for the polynomial coefficients (ignoring the regularizer for a moment).

$$G_i = \sum_J w_{iJ}\, \chi_J \chi_J^\top, \qquad \Theta_i = G_i^{-1} \sum_J w_{iJ}\, \chi_J u_J^\top, \qquad \chi_J = \chi_k(z_J)$$

If we pull out the constant coefficient with $e_0$ then the output collapses back to a weighted sum of values.

$$o_i^{(n,k)} = \sum_J a_{iJ}^{(k)}\, u_J, \qquad a_{iJ}^{(k)} = w_{iJ}\, e_0^\top G_i^{-1} \chi_k(z_J)$$

Notably, the weights $a_{iJ}^{(k)}$ are signed -> they are not probabilities, and that is precisely the point. Instead, they satisfy moment conditions that say exactly what the estimator reproduces, holding whenever $G_i$ is invertible.

$$\sum_J a_{iJ}^{(k)} = 1, \qquad \sum_J a_{iJ}^{(k)}\, z_J^{\otimes r} = 0 \quad (1 \le r \le k)$$

The first condition preserves mass and the rest cancel every local moment up to order $k$. At $k = 0$ you are only mass-preserving and you are back at softmax. At $k = 1$ you also cancel the first moment, which is LLA and Parallax. At $k = 2$ you cancel curvature, and so on. Degree $k$ estimates the local $k$-jet of the value function.

Learn the probe instead of solving

The exact fit is hopeless at scale, since each query needs its own solve over monomial features and the feature dimension blows up fast. Parallax's insight is to stop solving and learn the probe directly instead.

$$o_i^{\textsf{PLX}} = o_i^{\textsf{SA}} - \Sigma_{KV}^{(i)} \rho_i, \qquad \rho_i = W_R x_i$$

The same idea lifts to higher degree. We build centered polynomial statistics and contract them against a learned probe. Note that the probe carries the sign, so this reduces to Parallax at $(1,1)$.

$$o_i^{(n,k)\text{-PLX}} = \mathbb{E}_{p_i}[u_J] + \operatorname{Cov}_{p_i}\big(u_J, \tau_k(\phi_J)\big)\, \theta_i, \qquad \theta_i = R_k(x_i)$$ $$o_i = \mathbb{E}_{p_i}[u_J] + \sum_{r=1}^{k} \operatorname{Cov}_{p_i}\big(u_J, \phi_J^{\otimes r}\big)\, \theta_{i,r}$$

At $k = 2$ this is two corrections stacked on softmax: the first is linear Parallax and the second is a learned curvature probe.

$$o_i = \mathbb{E}[u_J] + \operatorname{Cov}(u_J, \phi_J)\, \rho_i + \operatorname{Cov}(u_J, \phi_J^{\otimes 2})\, H_i$$

The streaming form survives

For real kernels you never want $G_i^{-1}$. Instead, you want the moment form, because it streams over KV blocks the way Parallax does, with extra branches but no extra I/O. To do this, we keep running accumulators for the weights, the value sum, and each polynomial moment.

$$D = \sum_J w_J, \quad O = \sum_J w_J u_J, \quad M_r = \sum_J w_J\, \phi_J^{\otimes r}, \quad C_r = \sum_J w_J\, u_J \otimes \phi_J^{\otimes r}$$

The covariance at order $r$ is the cross moment minus the outer product of the means.

$$\operatorname{Cov}(u, \phi^{\otimes r}) = \frac{C_r}{D} - \frac{O}{D} \otimes \frac{M_r}{D}$$

The forward is then the value mean plus the probed corrections, with one per order.

$$o_i = \frac{O}{D} + \sum_{r=1}^{k} \left( \frac{C_r}{D} - \frac{O}{D} \otimes \frac{M_r}{D} \right) \theta_{i,r}$$

The exact $\theta_{i,r}$ is a full order-$r$ tensor.

Example: 2-simplicial LLA

For conreteness let us consider the case of $(2,1)$. 2-simplicial LLA/Parallax is ordinary LLA/Parallax with token-pairs as the datapoints. We lift each pair to one key and one value through a Hadamard product.

$$\phi_{jk} = k_j \odot k'_k, \qquad u_{jk} = v_j \odot v'_k$$ $$o_i^{\textsf{2LLA}} = o_i^{\textsf{2SA}} - (1 + \eta_i)\, \Sigma_{U\Phi}^{(i)} \rho_i^\star, \qquad \rho_i^\star = \Sigma_i^{-1} \mu_i$$

We replace the per-query solve with a learned probe and drop the boundary factor, exactly as Parallax does.

$$o_i^{\textsf{2PLX}} = o_i^{\textsf{2SA}} - \Sigma_{U\Phi}^{(i)} W_R x_i$$ $$o_i^{\textsf{2PLX}} = \sum_{(j,k) \in \mathcal{C}_i} s_{ijk}\, u_{jk}, \qquad s_{ijk} = p_{ijk}(1 - a_{ijk} + \bar a_i), \qquad \sum_{j,k} s_{ijk} = 1$$

Ordinary LLA corrects boundary bias in the local cloud of tokens, whereas 2-simplicial LLA/Parallax corrects it in the local cloud of token-pairs. It's the same regression, but with richer datapoints.

Why this design space

Both knobs do the same thing for different reasons. They change how the regressor behaves outside the data it has already seen.

Softmax attention is stuck inside the convex hull of its values. The output is a convex combination, so it can interpolate but never extrapolate. Both axes break out of the hull, in different ways.

The simplicial axis expands the hull itself. A lifted datapoint $\phi_{jk} = k_j \odot k'_k$ is a product of keys, so it generally lands somewhere the original keys never reach. Adding pairs and triples enriches the cloud with points outside the original convex hull, and the regressor gets new directions to retrieve along.

The polynomial axis changes the shape of the fit at the boundary. A local constant flattens out past the edge of the data, which is the boundary bias. A local linear fit carries the local slope outward, and a local quadratic carries curvature. Higher degree extrapolates more aggressively, for better or worse.

What explodes

The number of simplices per query grows polynomially in context length, and the monomial dimension grows fast in the degree once the lifted features have width $m$.

$$|\mathcal{C}_i^{(n)}| = O(i^n), \qquad \dim\big(\phi_J^{\otimes k}\big) = \binom{m + k}{k}$$

The exact estimator is feasible only for tiny $n$, $k$, and feature width. There are only a few cases that are actually tractable, such as the $(2,1)$ case. There are ways to overcome this barrier through approximation, but we will not explore that here.

Exploring the landscape

$(2,1)$ gives pairwise relational retrieval with first-order debiasing and $(1,2)$ gives ordinary attention with a curvature correction. Full $(2,2)$ is the next step which makes the most sense, though will require careful kernel desing.
