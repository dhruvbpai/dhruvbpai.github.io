Capacity vs Mixing in Associative Memory

May 2026

Every sequence mixer can be viewed as an associative memory. You store $N$ key-value pairs $(k_i, v_i)$ and retrieve values given queries. The Universal Hopfield Network [1] makes this precise. You pick a similarity function $\operatorname{sim}(q, k_i)$ and a separation function $\operatorname{sep}$ that turns similarities into weights.

$$y(q) = \sum_{i=1}^{N} \operatorname{sep}\!\big(\operatorname{sim}(q, k_i)\big) \, v_i$$

Set $\operatorname{sim} = q^\top k$ and $\operatorname{sep} = \operatorname{softmax}$. You get attention. Set $\operatorname{sep} = \text{id}$ and you get linear attention. Set $\operatorname{sep} = \text{ReLU}$ and you get the kernel FFN layers use. The choice of kernel is the whole game.

$$\text{Attn}(q, K, V) = \sum_{i=1}^{N} \frac{\exp(\beta \, q^\top k_i)}{\sum_j \exp(\beta \, q^\top k_j)} \, v_i$$

Softmax attention is a single-step Hopfield update [1] with energy

$$E(q) = -\frac{1}{\beta} \log \sum_j \exp\!\left(\beta \, q^\top k_j\right)$$

Stored patterns sit at energy minima in the energy landscape.

β = 2.0

What happens at finite $\beta$? Ramsauer et al. [1] identify three regimes. Low $\beta$ averages globally over all stored patterns. High $\beta$ converges to a single fixed point. The interesting regime is in between, where the system settles into metastable states that average over subsets of nearby patterns.

Smoothing is the feature

The standard question about an associative memory is capacity. How many pairs can you store and retrieve exactly? For linear attention [3], the memory is $S = \sum_i v_i k_i^\top$ and querying decomposes into signal plus interference.

$$Sq = \underbrace{v_j (k_j^\top k_j)}_{\text{signal}} + \underbrace{\sum_{i \neq j} v_i (k_i^\top k_j)}_{\text{noise}}$$

Exact retrieval requires orthogonal keys, limiting capacity to $N \leq d_k$. JL-style arguments push this to $\sim 2^{\epsilon d_k}$ at $\epsilon$-error. These bounds all assume you want near-exact retrieval, but attention almost never retrieves exactly.

β = 0.5

Transformers could learn exact retrieval by sending $\|q\|_2 \to \infty$. In practice they don't. Attention maps show smooth distributions over keys. The operating regime that makes most heads useful is metastable.

Zhong et al. [2] provide a useful lens to compare the theoretical capacity of different associative memories. The separation function $\kappa$ determines a retrieval SNR.

$$\text{SNR}_\kappa^{-1} = N \cdot \frac{\mathbb{E}_{k_j}\!\left[\kappa^2(k_j, k_i)\right]}{\kappa^2(k_i, k_i)}$$

This measures how much a random key leaks into the target's retrieval.

Kernel $\kappa$Used by$\text{SNR}^{-1}$Character
LinearLinear attention$N / d_k$Low precision, maximal superposition
ReLUFFN layers$\sim N / d_k^{1/2}$Intermediate precision, polysemantic storage
ExponentialSoftmax attention$\sim N \cdot e^{-2d_k}$High precision, monosemantic slots

Different kernels are different points on the precision-density spectrum. The exp kernel gives clean, one-fact-per-slot storage, while the ReLU kernel packs more knowledge into fewer parameters through polysemantic superposition. However, the implicit assumption is that the soft regime is a compromise. Lower precision is the necessary cost of denser storage, and the ideal is still exact retrieval. I believe that framing is incomplete.

Crosstalk vs interpolation

Not all interference is created equal. When a soft memory retrieves a blend of values, two very different things could be happening.

The first is crosstalk. Keys collide in the kernel's similarity metric but their values are semantically unrelated (or one value should take precedence over the other). This is precisely the interference that capacity analysis measures and that delta-rule updates [4] try to correct. If $k_a \approx k_b$ but $v_a$ and $v_b$ point in unrelated directions, then any query near $k_a$ retrieves a corrupted signal.

$$\hat{v} \approx \alpha \, v_a + (1 - \alpha) \, v_b$$

The $v_b$ term is pure error.

The second is semantic interpolation. Keys are nearby because their values are genuinely related. The values lie on a locally smooth semantic manifold.

$$v_i \approx f(k_i)$$

In this case, the weighted average is not failed retrieval but rather a useful smoothing. The output estimates $f(q)$ from a neighborhood of stored samples, and the smoothing can actually improve the estimate by averaging out noise or providing further contextual information.

The same mathematical operation, but different alignment between the kernel's notion of proximity and the structure of the stored values.

Left: interpolation -> aligned values blend coherently. Right: crosstalk -> conflicting values produce noise.

The kernel regression view makes this precise. Softmax attention is an instance of the classical Nadaraya-Watson estimator from nonparametric statistics. The bias of this estimator is small when $f$ is smooth on the kernel's support. The variance decreases with more samples in the neighborhood. Soft retrieval over a value-coherent neighborhood does not sacrifice accuracy, but rather improves it.

Smooth basins, high walls

The energy landscape makes the desiderata concrete. A good associative memory needs two things simultaneously: smooth basins within semantic regions, so related memories interpolate, and high energy barriers between incompatible regions, so unrelated values never blend.

We can formalize some version of this. Partition the stored patterns into semantic clusters $\{C_1, \ldots, C_m\}$. Within each cluster, values are compatible (blending is useful) and across clusters, values conflict (blending is crosstalk). A good energy landscape satisfies two conditions.

Intra-cluster smoothness. For patterns $k_a, k_b$ in the same cluster, the energy surface should be smooth between them. If $E$ is $L$-Lipschitz on the geodesic connecting $k_a$ and $k_b$, then

$$|E(q) - E(q')| \leq L \|q - q'\| \quad \text{for } q, q' \in \operatorname{conv}(C_s)$$

Small $L$ means the basin floor is locally flat. A query anywhere in the cluster's convex hull retrieves a smooth blend of the cluster's values. This is the regime where metastable averaging is useful.

Inter-cluster separation. Between clusters, the energy barrier should be large. Define the barrier height between clusters $C_s$ and $C_t$ as

$$B(C_s, C_t) = \min_{q \in \text{path}(C_s, C_t)} E(q) - \max\!\big(E^*(C_s),\, E^*(C_t)\big)$$

where $E^*(C_s)$ is the minimum energy within cluster $s$. Large $B$ means a query must climb a significant energy hill to cross from one basin to another. The softmax distribution suppresses contributions from patterns on the far side of a barrier exponentially in $\beta \cdot B$.

Learned energy landscape with smooth basins and high inter-cluster barriers. Drag to rotate.

What makes softmax attention powerful here is that it can interpolate continuously between totally flat and totally spiky. At $\beta = 0$ the landscape is a constant and every pattern contributes equally. As $\beta$ grows, basins deepen and barriers sharpen until at $\beta \to \infty$ each pattern is an isolated delta. The learned $\beta = ||q||_2 ||k||_2/\sqrt{d_k}$ sits in the middle, and the learned key geometry determines which regions of the landscape are smooth and which have walls. This is a much richer parameterization than a fixed kernel width.

End-to-end learning shapes this landscape. The model does not just learn what to store. It learns where to put the walls. Training carves valleys around clusters of compatible memories and raises barriers between incompatible ones. The kernel, key-value geometry, and temperature all co-adapt.

The Anthropic superposition work [5] makes a related point. Models represent more features than they have dimensions by tolerating interference. That interference is not a defect but rather a necessary mechanism to achieve compression. The real question is whether the architecture gives you enough control over which cross-terms survive.

Put another way, interference is only noise relative to an exact-recall objective. Relative to abstraction, generalization, and composition, controlled interference is the computation.

Architectural choices as landscape shaping

This perspective makes concrete predictions about why certain architectural choices work. I selected, somewhat arbitrarily, three recent developments that illustrate the principles - fine-grained mixture-of-experts routing, sparse attention under heavy key-value sharing, and channel-wise forget gates in linear attention.

Fine-grained MoE. A mixture-of-experts layer partitions the associative memory into $E$ sub-memories (experts), with a router $g(x) \in \Delta^E$ selecting a sparse subset per token. The output is

$$y = \sum_{e \in \text{TopK}(g(x))} g_e(x) \cdot f_e(x)$$

Through the energy landscape lens, routing erects hard barriers between expert basins. Tokens assigned to expert $e$ never interfere with memories stored in expert $e'$. Within each expert, the FFN kernel still performs smooth mixing over its stored associations. The architecture thus decouples two scales of the landscape -> coarse-grained selectivity across experts (suppressing crosstalk between unrelated knowledge clusters) and fine-grained interpolation within each expert (preserving useful superposition among related facts). Making experts smaller and more numerous as in fine-grained MoE sharpens the inter-basin barriers while keeping intra-basin smoothness.

Fine-grained expert segmentation [9]. Splitting $N$ experts into $mN$ smaller ones and activating $mK$ increases combinatorial flexibility while keeping compute constant.

Sparse attention under GQA. Grouped-query attention shares each key-value head across $G$ query heads. As $G$ grows, the keys must serve increasingly diverse queries, and the pressure on the associative memory grows. The energy landscape flattens as margins $s(q, k_{i^*}) - s(q, k_j)$ shrink, and the exponential kernel can no longer separate signal from interference.

$$y = \underbrace{p_{i^*} v_{i^*}}_{\text{signal}} + \underbrace{\sum_{j \neq i^*} p_j \, v_j}_{\text{crosstalk that grows as margins shrink}}$$

In our work Sparsity is Cool [7], we analyzed this phenomenon. Sparse attention zeros out the low-relevance terms entirely by selecting only the top-scoring key blocks per query. In the energy landscape, this erects infinite barriers around irrelevant basins, restricting retrieval to a neighborhood where the kernel can still discriminate.

Sparse attention (MoBA, NSA) outperforms dense attention most strongly under heavy KV sharing. Data from [7].

Under MQA (1 KV head, 16 query heads), both MoBA and NSA beat the dense transformer by a wide margin, while under MHA (16 KV heads) the gap narrows because head-specific keys already provide well-separated basins. The t-SNE geometry of key vectors confirms the mechanism -> under MQA, the key cloud is nearly isotropic and queries must snake around it, making sparse block selection the means by which heads achieve diverse retrieval behavior despite sharing a single key set.

Channel-wise forget gates and the delta rule. In recurrent associative memories, the state $S_t \in \mathbb{R}^{d_k \times d_v}$ accumulates key-value associations over time. The simplest update is additive Hebbian learning rule.

$$S_t = S_{t-1} + v_t k_t^\top$$

This is linear attention's outer-product accumulation [3]. Every new association adds to the existing superposition, which monotonically increases interference. The delta rule [4] corrects existing associations before writing:

$$S_t = S_{t-1} + \beta_t(v_t - S_{t-1} k_t) k_t^\top$$

The correction term $-\beta_t S_{t-1} k_t k_t^\top$ is a rank-one update that evicts the current value stored at $k_t$ before writing the new one. In the energy landscape, this deepens the basin around $k_t$ by eliminating the crosstalk residual from stale associations.

KDA [6] and RWKV-7 [8] extend this further by replacing the scalar decay with a diagonal forget gate $\Lambda_t = \text{diag}(\alpha_t)$ where $\alpha_t \in [0,1]^{d_k}$:

$$S_t = (\Lambda_t \otimes I_{d_v}) \, S_{t-1} + \beta_t v_t k_t^\top$$

Each feature dimension now has an independent forgetting rate. This is a much richer operation on the energy landscape than scalar decay. Dimensions encoding transient context (syntactic role, local position) can decay rapidly, collapsing those axes of the landscape and sharpening retrieval along them. Dimensions encoding persistent semantics can decay slowly, maintaining broad basins that support interpolation. The channel-wise gate gives the model per-dimension control over the barrier height in the memory's energy surface, which can help explain why diagonal forget gates achieve so much stronger empirical results than their scalar counterparts.

Architectural advances that improve language modeling tend to give the memory finer-grained control over which cross-terms to preserve and which to suppress, in an end-to-end learnable fashion. Exact-recall capacity is a special case, but not necessarily the operating regime that matters. What matters is the ability to shape the landscape so that the right things blend and the wrong things don't.
